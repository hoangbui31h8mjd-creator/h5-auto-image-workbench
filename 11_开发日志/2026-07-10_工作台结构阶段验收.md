## 下一步建议

1. 先做 `packages/agent-contract`，把 `AgentEvent`、Run 状态、ProducedFile、NextAction 等从前端 state 中抽成共享契约。
2. 在 `apps/daemon` 做 mock event stream，让前端从本地 mock state 迁移到「Frontend Agent panel -> daemon -> event stream」结构。
3. 再设计旧 Python runner 的适配层，只抽取切片计划、分段生图、拼接长图和质检所需能力，不搬旧项目整体。
4. 为项目文件区设计真实本地落盘路径，包括 `brief.md`、`h5-artifact.json`、`prototype.html`、`slice-plan.json`、`long-image.png`、`export.zip`。
5. 接入真实 runner 前，不要继续扩展账号、团队协作、模板市场、自由画布等非第一阶段能力。
