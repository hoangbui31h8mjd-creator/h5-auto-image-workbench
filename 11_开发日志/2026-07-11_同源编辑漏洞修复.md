# 2026-07-11 同源编辑漏洞修复（18.1）

## 范围

本轮只修复 legacy runner 正式原型的同源编辑漏洞，不进入第十九步，不接真实生图，不运行 `run_local_h5.py`，不改变「一级 Home + 二级 Project Workspace」结构。

## 实际修复

### 1. legacy 素材操作在写入前阻断

- `ProjectStore.saveAsset()` 与新增的 `ProjectStore.clearAsset()` 会先检查 `prototypeSource.kind`。
- 当正式原型来源为 `legacy_runner` 时，上传、替换、清除一律在任何项目目录写入前抛出 `RequiresRegenerationError`。
- `server.ts` 将该错误统一返回为 HTTP `409`，结构化字段为 `error.code = "requires_regeneration"`。
- `AssetSlotsView` 在 legacy 来源时禁用上传、替换、清除，并显示「需要重新生成候选原型」的原因。
- `artifact_renderer` 原型仍允许正常本地素材上传；Home 在生成 legacy 候选前上传 KV 的路径未改动。

### 2. legacy 文案定位不再用 `.first()` 掩盖歧义

- `legacyRunnerRenderer.findTextTarget()` 先收集全部候选节点，再判断数量。
- 显式 `data-h5-field`、`data-field`、`data-text-field` 命中重复时全部拒绝。
- 标题和正文回退选择器命中 0 个或超过 1 个时，均返回 `requires_regeneration`。
- 只有候选节点恰好为 1 时，才会写入文本；拒绝时正式 `prototype.html` 与 `h5-artifact.json` 不变。

### 3. 高度以归一化 Artifact 为唯一来源

- `legacyRunnerRenderer` 不再使用未经约束的 `patch.value`。
- 它改为读取 `artifactAfter.sections[].height` 后同时写入 `data-workbench-height` 和 scoped CSS 的 `min-height`。
- 因此输入 `50` 时 Artifact 与 HTML 都为 `120`；输入 `5000` 时两者都为 `1800`。

### 4. 当前状态文档去歧义

- `00_当前开发状态.md` 现将「产品能力权威状态」与「当前打开项目状态」分开记录。
- 第二轮自审前重新读取 daemon 与 `current/project.json`：active current 为 `daemon_project_1783706318489`、4 个 section、`artifact_renderer`、`draft`，`kv_main` 已上传至 `assets/kv_main.jpg`。该项目在 01:58 已创建，早于本轮自审。
- 9 模块、`legacy_runner`、KV 为空的项目明确标记为历史 snapshot：`projects/snapshots/daemon_project_1783649876109/1783701461779-033289aa-6b07-4022-bf28-23330a129236/`，不是 current。

## 第二轮对抗式自审

### 自审发现并修复的问题

1. 第一轮测试仍以简化 3 模块 HTML 为主，缺少真实 9 模块结构：新增从历史 snapshot 只读复制到 `mkdtemp()` 的独立 fixture 测试；原 snapshot 前后哈希一致。
2. `saveAsset()` 先写素材、再逐个写 JSON，受控磁盘失败时可能留下半套状态；`patchArtifact()` 的旧回滚也未覆盖所有派生文件与旧备份。现改为整组文件先暂存、再提交，失败按文件快照完整恢复。
3. Web `apiJson()` 丢失 daemon 的结构化 `error.code`，Inspector 依赖字符串判断。现新增 `DaemonApiError`，保留 HTTP status / code / details，并用结构化 code 显示重生成提示。
4. 素材 PUT / DELETE 缺少持久化队列屏障，可能与上一轮异步状态写入交叉。现与 Artifact PATCH 一样，变更前先 `flushPersistenceQueues()`。
5. 第一轮日志记录的 active current 已过期。第二轮先读取 API、磁盘和 snapshot 时间线，再改为当前真实项目 `daemon_project_1783706318489`。
