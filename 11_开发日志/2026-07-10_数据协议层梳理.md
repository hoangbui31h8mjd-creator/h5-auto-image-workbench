# 2026-07-10 数据协议层梳理

## 本次修改背景

第一阶段结构验收已经完成。当前前端已经具备一级 Home、二级 Project Workspace、左侧 Agent 面板、右侧设计文件、原型预览、生成长图、素材/KV 等区域。

本轮目标是进入第二步：在不接真实 Codex CLI、不接真实 runner、不重做 UI 的前提下，先把未来接 daemon / CLI / 生图服务所需的数据协议层落地。

## 本次目标

- 定义前端与 daemon 之间稳定传输的 `AgentEvent`。
- 扩展 `H5Artifact` 协议，使结构化原型能承载 H5 类型、设计规范、模块字段和全局图片槽位。
- 定义右侧设计文件区需要的 `FileManifest`。
- 定义一次 H5 生产任务的 `RunState`。
- 定义素材/KV 页面与 H5Artifact 共用的 `ImageSlot`。
- 新增统一 `mockProjectState`，让当前前端先从统一 mock 状态取数。
- 保持当前 UI 结构不变，不接真实 CLI 和真实 runner。
