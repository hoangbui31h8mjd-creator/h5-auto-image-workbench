# 2026-07-10 基础人工编辑能力

## 本次修改背景

第十一步已经解决了 Artifact draft 应用后前端预览不刷新的问题。现在进入第十二步，需要让设计师在原型预览阶段做最小人工编辑，并把修改保存回正式 `h5-artifact.json`。

本轮继续保持产品边界：不做完整 Figma，不做自由画布，不接真实生图，不重做 UI。

## 本次目标

- 在 daemon 增加 artifact patch API。
- 在 Inspector 中支持最小人工编辑。
- 保存后写回正式 `h5-artifact.json`，并重新生成 `prototype.html`。
- 前端保存成功后刷新当前项目 state，原型预览无需手动刷新即可更新。

## 新增能力

- `PATCH /api/projects/current/artifact` 支持：
  - `update_artifact_title`
  - `update_section_text`
  - `update_section_prop`
  - `update_section_height`
  - `move_section_up`
  - `move_section_down`
  - `hide_section`
  - `restore_section`
- 保存前会校验 section 存在、字段合法、至少保留一个可见 section。
- 保存前会备份上一版正式 artifact 到 `h5-artifact.previous.json`。
- 保存后会重新生成 `prototype.html`。
- 保存后会写入 AgentEvent：`assistant_message`、`file_written`、`artifact_updated`。

## 交互变化

- 进入「原型预览」tab 后，右侧 Inspector 不再只是只读。
- 设计师可以编辑 artifact 标题。
- 设计师可以编辑当前模块的 textFields。
- 设计师可以编辑当前模块 props 中的 string / number 字段。
- 设计师可以调整模块高度。
- 设计师可以上移 / 下移当前模块。
- 设计师可以隐藏 / 恢复当前模块。
- 点击「保存修改」后，前端会调用 daemon patch API；保存成功后重新拉取当前项目状态。
