# 内部执行手册

> 注意：本文件来自旧 Codex 产品包，仅作为旧本地 runner 调用方式参考。

本文件只给 Codex 或维护者使用，不是普通用户入口。

普通用户只需要看 `../开始使用.md`，并在对话里提供头图和活动内容。

## 标准化输入

当用户在对话中直接提供素材时，Codex 应自动整理到：

```text
../01_用户输入区/当前任务/
```

建议内部标准文件：

```text
kv.jpg 或 kv.png
content.md
special_requirements.md  # 可选
```

说明：

- 用户不需要知道这些文件名。
- `content.md` 用来沉淀用户提供的活动内容。
- `special_requirements.md` 只记录用户明确提出的特殊要求。
- 默认语言为巴西葡语 `pt-BR`，用户明确指定其他语言时才覆盖。
- 头图和活动内容必须同时存在；缺任意一项时不要运行，不要用历史案例代替。
- 如果命令返回 `Input incomplete`，应回到对话中向用户补齐输入，而不是修改代码或继续套样例。

## 跑真实生图

真实生图时，不要把 API key 写入文件。推荐让用户在终端临时输入环境变量。

```bash

.venv/bin/python -m h5_agent.runner.run_local_h5 \
  --task-dir "/Users/yiming/Desktop/h5自动生图项目文档（正在跑）/01_当前工作区/01_H5_Agent产品包/01_用户输入区/当前任务" \
```

## 维护规则

- 不要把测试样例当成默认任务。
- 不要把 API key 写入任何文档、样例、日志或输出。
- 产品包只接收已经在研发工作区验证过的稳定规则。
- 实验、失败案例和临时调试先放研发工作区。
- 产品包的 `01_用户输入区/当前任务/` 和 `05_输出结果/` 发布时应保持空壳，只保留说明文件。
