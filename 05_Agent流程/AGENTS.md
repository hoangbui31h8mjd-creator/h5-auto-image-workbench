# H5 Agent Product Instructions

你现在位于“商单 H5 自动生成 Agent”的产品包根目录。

当用户说“帮我生成一张 H5”“做一张活动长图”“生成 H5 原型”时，按下面规则直接执行。

## 用户只需要给什么

1. 头图或 KV 图。
2. 活动内容：活动名称、玩法、奖励、规则、目标用户，或希望出现的页面板块。

如果缺头图，只问头图；如果缺活动内容，只问活动内容。不要要求用户理解工程文件、Dify、ComfyUI、runner 或配置参数。

如果两项输入没有齐全，必须停下来向用户补问，不得套用历史 PUBG 案例、参考案例或默认文案继续生成。即使本地命令被误触发，只要返回 `Input incomplete`，也应把它理解为“需要补输入”，而不是程序故障。

## 默认产品规则

- 默认语言：巴西葡语 `pt-BR`。
- 只有用户明确指定英文、中文、西语等其他语言时，才覆盖默认语言。
- 用户给的头图/KV 默认就是 H5 第一屏主视觉，必须嵌入 HTML 原型顶部。
- KV 不要重复铺到后续模块；后续模块只延展 KV 的风格、颜色、材质和氛围。
- 页面结构、组件、切图、参考带、拼接和质检规则由本包 Skill 自动处理。

## 必读入口

执行前先读：

1. `00_AGENT入口说明.md`
2. `02_执行Skills/skill_manifest.json`
3. `02_执行Skills/README.md`
4. `03_知识库/H5原型生成知识库_单文件版.md`
5. `_agent_internal/执行手册.md`

如果用户只是在询问怎么用，优先回答 `开始使用.md` 里的用户侧说明。

## 输出原则

优先交付：

- 完整原型 HTML。
- 完整原型图。
- 原型切片图。
- 最终 H5 长图或 mock 结果。
- 简短质检结论。

不要把历史案例当作本次任务自动运行；`90_参考案例_不自动运行/` 只供参考。

产品包默认不自带可执行任务素材，也不自带历史输出。每一次生成都应从用户本次提供的头图和活动内容开始。
