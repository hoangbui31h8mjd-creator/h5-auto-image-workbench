# 文件更新与归档规则

## 当前区原则

`01_当前工作区/` 只放当前阶段正在跑的 Codex 本地 H5 Agent。

允许存在：

```text
00_当前阶段必读/
01_H5_Agent产品包/
02_Codex本地Agent工作区/
```

不再放：

```text
Dify 工作流目录
ImageService 图像服务目录
ComfyUI 工作流目录
旧知识库导入目录
旧演示素材目录
```

这些内容统一进入历史版本或交付展示。

## 更新规则

1. 任何新规则、新 Skill、新提示词、新流程，先在 `02_Codex本地Agent工作区/` 修改和验证。
2. 草稿、实验、失败案例、调试输出只放 `02_Codex本地Agent工作区/`。
3. 确认稳定后，再从工作区同步到 `01_H5_Agent产品包/`。
4. 产品包中不能放会被默认执行的历史案例。
5. 缺 KV 或内容信息时，入口规则必须要求 Codex 补问；特殊要求不是必填项。
6. API key、密码、临时 tunnel 地址不得写入文档或样例。


运行方式：


脚本只复制，不删除当前工作区。

## 提交前检查

1. 根目录 `00_项目入口_README.md` 是否指向正确入口。
2. `01_H5_Agent产品包/开始使用.md` 是否仍是用户第一入口。
3. `01_H5_Agent产品包/00_AGENT入口说明.md` 是否仍是 Codex 执行入口。
4. 产品包中是否误放了历史任务或测试输出。
5. 工作区更新是否经过验证后再同步到了产品包。
6. 文档路径是否跟移动后的目录一致。
7. 是否明确区分“已验证”和“待验证”。
