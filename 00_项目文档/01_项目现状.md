# 项目现状

## 当前阶段

当前阶段是：

```text
Codex 本地 H5 Agent 产品包
```

也就是说，项目不再把 Dify 和 ComfyUI 画布作为产品主入口，而是转向一个可以被用户直接用 Codex 打开的本地 Agent 项目包。

用户目标体验：

```text
用户打开 01_H5_Agent产品包
-> 说“帮我生成一张 H5”
-> Codex 检查 KV、内容信息
-> 缺信息就补问
-> 信息齐全后生成原型、切片、生图、拼接和结果
```

## 已完成

- 将产品包和研发工作区拆开。
- 产品包中保留执行 Skill、知识库、流程说明、用户输入区和输出区。
- 入口规则已明确：缺 KV 或内容信息时必须先问用户；默认语言为巴西葡语 `pt-BR`，特殊要求才由用户补充。
- 历史 PUBG 样例不再作为默认任务。
- 旧 Dify/ComfyUI 验证链路从当前区移入历史归档。

## 当前还没完成

- 产品包仍依赖代码项目根目录里的本地 runner。
- 真实生图 API key 仍需用户在终端或运行时提供。
- 还没有做成完全独立的一键应用。
- 原型质量、切图质量、生图提示词和拼接质量还需要继续通过真实任务打磨。


```text
../02_Codex本地Agent工作区/
```
