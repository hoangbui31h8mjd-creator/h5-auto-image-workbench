> Figma 源文件：https://www.figma.com/design/sTYDbNPI1U3xTeg5FXVL9b/%E7%A7%AF%E6%9C%A8%E7%BB%84%E4%BB%B6

## 全局基础规范

| 属性 | 值 | 说明 |
|------|-----|------|
| 画布宽度 | `828px` | 所有组件统一，适配移动端 H5 |
| 内容区宽度 | `748px` | 左右各 40px 安全边距 |
| 安全边距 | `40px` | 组件内容距边框最小间距 |
| 组件间距 | `24px ~ 32px` | 同页面内组件垂直间距 |
| 卡片圆角 | rounded-rectangle | 非直角，具体值见各组件 |
| 列表行高 | `128px`（排行榜）/ `168px`（任务列表） | 保持视觉节奏一致 |
| 进度条高度 | `8px` | 圆角处理，底色与进度色分层 |
| 弹层规格 | `828×1520` 或 `828×1792` | 标准 H5 全屏弹层 |

**按钮高度规范（5档）：** `56px` · `64px` · `80px` · `88px` · `96px`，根据场景重要程度选用。

**按钮动效规范（30fps 缓入缓出）：**
按下 scale(0.85) → 弹起 scale(1.1) → 回落 scale(1.0)
时间节点：0s 按下 · 0.04s 最小 · 0.12s 最大 · 0.16s 回弹 · 1.0s 静置

## 原型图生成规则（使用 show_widget 时必读）

**禁止调用 `read_me` 工具。** `read_me` 会加载 claude.ai 自身的 380px 设计系统，与积木 828px 规范冲突，导致尺寸全部错乱。

原型图用 `show_widget` 直接输出原生 HTML + 内联 CSS，不引入任何 claude.ai CSS 变量（不用 `--color-background-primary` 等）。

**标准模板结构：**
```html
<style>
*{box-sizing:border-box;margin:0;padding:0;font-family:Arial,sans-serif}
.w{width:828px;background:#f0f0f0}
.pad{padding:24px 40px 32px}   /* 内容区左右默认 40px，用户要求时可调整 */
/* 其余样式按积木规范写 px 值 */
</style>
<div class="w">
  <!-- 组件按规范尺寸堆叠 -->
</div>
```

**原型图风格：** 黑白灰色块占位，`#e8e8e8` 填充图片/视频区，`#f5f5f5` 填充按钮/背景，`1px solid #e0e0e0` 边框，文字用真实内容。不加任何装饰色、渐变、图标装饰。

## 通用设计原则

1. 所有组件画布宽度统一 **828px**，内容区 **748px**（左右各 40px 边距）
2. 卡片圆角统一使用 `rounded-rectangle`（非直角）
3. 按钮高度五档：`56` · `64` · `80` · `88` · `96px`，按重要程度选用
4. 头像/图片均使用圆角矩形或正圆，避免直角裁切
5. 列表行高统一：排行榜 **128px**，任务列表 **168px**
6. 进度条高度统一 **8px**，圆角，底色与进度色分层
7. 弹层规格：标准 **828×1520** 或 **828×1792**
8. 颜色、文字颜色、按钮底色等均设计为**可配置**，以适应不同活动主题
9. 支持 RTL（阿拉伯语等）语言排版，需单独验证
10. 按钮动效统一：30fps 缓入缓出，scale 0.85→1.1→1.0
