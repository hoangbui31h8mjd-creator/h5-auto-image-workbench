# H5 Agent 产品包发布前验收记录 2026-07-09

## 验收目标

本轮按正式发布前口径检查：先在研发工作区修正规则和代码，再同步到产品包。核心目标是让新 Codex 打开产品包后，不需要先理解历史项目，也不会误用旧案例，用户只要提供头图和活动内容即可开始生成。

## 本轮修正

1. 输入门槛收紧：头图/KV 与活动内容必须同时存在；缺任意一项时停止并向用户补问，不再自动套用 PUBG 或历史样例。
2. 产品包清理：发布包默认不预置任务素材，也不预置历史输出；旧测试输入和旧输出转移到研发工作区归档。
3. 入口规则补齐：研发区的 `产品包AGENTS发布源.md` 与产品包 `AGENTS.md` 同步，明确新 Codex 的执行边界。
4. 默认语言修正：默认从第一步生成巴西葡语 `pt-BR`，不再先生成英文/中文后靠质检翻译。
5. 头图处理修正：用户给的 KV/头图默认作为 HTML 原型第一屏主视觉嵌入，后续模块只延展风格，不重复铺图。

## 三轮自查

### 1. 用户视角

结论：通过。

用户只需要提供两类东西：

- 头图或 KV 图。
- 活动内容，例如活动名称、玩法、奖励、规则、目标用户或希望出现的 H5 板块。

用户不需要填写技术要求文件、runner 参数、模型 endpoint、切图参数或工程配置。缺输入时，Agent 应只追问缺的那一项。

### 2. Codex 视角

结论：通过。

新 Codex 打开产品包后，应先读：

1. `AGENTS.md`
2. `00_AGENT入口说明.md`
3. `02_执行Skills/skill_manifest.json`
4. `03_知识库/H5原型生成知识库_单文件版.md`
5. `_agent_internal/执行手册.md`

如果输入不完整，不能运行历史案例；如果本地命令返回 `Input incomplete`，应回到对话中补问用户。

### 3. 维护视角

结论：通过。

- `01_H5_Agent产品包/`：稳定发布包，只放用户入口、执行 Skill、知识库、执行说明和空的输入/输出壳。
- `02_Codex本地Agent工作区/`：研发、测试、验收、失败案例和历史样例归档。
- `99_暂不启用与原始备份/`：不自动运行的历史素材和旧测试结果。

后续规则变更应先改研发工作区，验证通过后再同步产品包。

## 残留风险

1. 当前验收主要覆盖“原型生成 + 渲染 + 切片”，真实生图质量仍依赖 Image2/OpenRouter 的模型响应和提示词稳定性。
2. 图文混合知识库目前已能支撑基础规范，但公司内部 Figma/品牌规范的自动抽取仍需要继续完善。
3. 产品包目前适合有 Codex 的设计/产品同学本地使用；若要多人平台化使用，还需要服务端密钥管理、任务队列、素材权限和结果托管。
