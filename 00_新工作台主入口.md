# H5 Agent 工作台新项目主入口

本文件是新项目的当前主入口。

如果后续用 Codex、Claude、Cursor 或其他 Agent 打开本项目，应先读本文件，再读 PRD 和代码包说明。

## 1. 当前项目判断

旧项目已经验证了 H5 自动生成链路，但旧项目形态更接近：

```text
Codex 打开文件夹
-> 读取规则
-> 调用本地 runner
-> 输出一张 H5 长图和调试文件
```

新项目要转向：

```text
用户在工作台输入需求
-> Agent 补问缺失信息
-> 生成可编辑 H5 原型
-> 人在原型阶段修改
-> 确认后生图
-> 输出长图和项目包
-> 后续支持局部修改和版本管理
```

因此，旧项目是底层链路资产，不是新项目的产品入口。

## 2. 当前主线

第一阶段只做本地个人工具，不做多人协作。

核心闭环：

```text
需求输入
-> 生成原型
-> 人改原型
-> 生成长图
-> 导出
```

第一阶段优先级：

1. 定义 H5 Artifact 协议。
2. 做原型编辑工作台。
3. 接入旧 Python runner。
4. 接入 Codex / Agent 生成流程。
5. 保存生成记录和导出结果。
