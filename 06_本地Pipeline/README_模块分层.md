# 本地 Pipeline 模块分层

下一阶段不建议继续维护一个独立的 ImageService HTTP 中转服务，也不建议继续把 ComfyUI 当成产品核心。

推荐把能力拆成 Codex 本地可调用的 Python 模块。

## 推荐代码结构

代码根目录建议对应：

```text
h5_agent/
├── knowledge/     # 知识库和案例检索
├── prototype/     # 需求理解、蓝图、HTML 原型生成
├── render/        # HTML 渲染成完整原型图
├── slicing/       # 模块边界识别和智能切片
├── generation/    # Image2 分段生图
├── stitching/     # 参考带、重叠、羽化、拼接
├── storage/       # 任务目录、产物保存、结果索引
├── quality/       # 原型、切片、生图、拼接质检
└── runner/        # 一键运行入口
```

## 分层原则

每层都必须有明确输入和输出。

```text
输入层：KV、文案、要求
知识层：规则、案例、组件 Skill
蓝图层：页面模块和文案映射
原型层：HTML wireframe
渲染层：完整原型图
切片层：切点、切片图、裁切报告
生图层：分段图
拼接层：最终长图
质检层：错误、风险、重跑建议
归档层：任务文件夹
```

## 当前切片规则

```text
模型最大比例：1:3
默认画布：828px
参考带高度：80px
单张原型切片高度预算：当前原型图宽度 × 3 - 80
```

切点优先落在模块结束、子组件结束、自然留白处。不要固定切片数量，也不要为了切片数量切断主体内容。

详细见：

```text
切图与参考带规则_v0.1.md
```

## 为什么这样拆

- 方便在 Codex 本地直接调用。
- 方便单独测试每一层。
- 方便定位失败原因。
- 方便未来再封装成平台 Demo。

## 下一步最小实现

先跑通：

```text
KV + 文案 + 要求
-> 蓝图 JSON
-> HTML 原型
-> 完整原型图
-> 智能切片
-> 质检报告
```

这一步稳定后，再接真实 Image2 生图。
